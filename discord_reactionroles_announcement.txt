**NEW FEATURE: Reaction Roles System**

NexGuard now supports Reaction Roles! Users can automatically get or remove roles by reacting to messages with specific emojis.

**What Are Reaction Roles?**
Users react to messages with emojis to instantly receive roles. Remove the reaction to lose the role. Perfect for self-service role management without admin intervention.

**Setup Commands:**
- `/reaction-roles add` - Create a reaction role on any message
- `/reaction-roles remove` - Remove a specific reaction role
- `/reaction-roles list` - View all active reaction roles
- `/reaction-roles clear` - Remove all reaction roles from a message

**How to Set Up:**

1. Create or choose a message for role selection
2. Right-click the message and copy the Message ID
3. Configure reaction roles using:
```
/reaction-roles add message_id:[MESSAGE_ID] emoji:[EMOJI] role:[@ROLE]
```

**Example Setup:**
```
/reaction-roles add message_id:123456789 emoji:🎮 role:@Gamer
/reaction-roles add message_id:123456789 emoji:📚 role:@Student
/reaction-roles add message_id:123456789 emoji:🎨 role:@Artist
```

**Common Use Cases:**
- Game roles for different games
- Notification roles for announcements
- Interest-based community groups
- Access roles for special channels
- Color roles for display customization
- Pronoun roles for identification

**Key Features:**
- Instant role assignment and removal
- Respects Discord role hierarchy
- Multiple roles per message supported
- Persistent across bot restarts
- Automatic cleanup of deleted roles/messages
- Works with Unicode and custom emojis

**Permission Requirements:**
- Admins need "Manage Roles" permission to configure
- Bot must be able to assign target roles
- Target roles must be below bot's highest role

**Example Message:**
"React below to get your roles:
🎮 for Gamer role
📚 for Student role
🎨 for Artist role
🎵 for Music Lover role"

**Getting Started:**
1. Post a role selection message
2. Copy the message ID (Developer Mode required)
3. Use `/reaction-roles add` for each emoji/role pair
4. Bot automatically adds reactions to the message
5. Users can now react to receive roles instantly

**Management Tips:**
- Use `/reaction-roles list` to review configurations
- Pin role selection messages for easy access
- Group related roles on the same message
- Use clear, recognizable emojis
